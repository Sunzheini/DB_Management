Q: How would you scale this for production?
A: Connection pooling, read replicas, caching layer (Redis), partition
   large tables, migrate to PostgreSQL for advanced features, add monitoring
   with query performance tracking.

Q: How do you handle database migrations?
A: Version control schema changes, use migration tools (Alembic), maintain
   rollback scripts, test migrations on staging, apply during maintenance
   windows with blue-green deployment.

Q: What about database security in production?
A: Encrypted connections (TLS), encrypted at rest, principle of least
   privilege, separate read/write users, audit logging, regular security
   patches, no direct database access (API only).

Q: How do you monitor database performance?
A: Slow query logs, EXPLAIN ANALYZE on problem queries, index usage stats,
   connection pool metrics, deadlock detection, disk I/O monitoring,
   automated alerts for anomalies.
